# Animations

## Easing Functions

The variables for easing functions are all prefixed with `--ease-`. They're
designed to be used with the `animation-timing-function` property.

<div class="stack vertical">
  <AnimationExample easingFunctionName="default">
    A general-purpose easing function that works well for most use cases.
  </AnimationExample>
  <AnimationExample easingFunctionName="out">
    An easing function that starts fast and slows down at the end.
  </AnimationExample>
  <AnimationExample easingFunctionName="in">
    An easing function that starts slow and speeds up at the end.
  </AnimationExample>
  <AnimationExample easingFunctionName="both">
    An easing function that starts slow, speeds up in the middle, and slows back down at the end.
  </AnimationExample>
  <AnimationExample easingFunctionName="both-subtle">
    An easing function that is nearly linear but speeds up slightly in the middle.
  </AnimationExample>
  <AnimationExample easingFunctionName="spring">
    An easing function that moves slightly backwards at the beginning, moves quickly in the middle, and slightly overshoots the target state before settling at the end.
  </AnimationExample>
</div>

## Durations

The variables for durations are all prefixed with `--duration-`. They're
designed to be used with the `animation-duration` and `transition-duration`
properties.

<div class="stack vertical">
  <DurationExample name="s">
    Use this duration for small interactions that don't necessarily need to be
    animated, but look better with a subtle animation.
  </DurationExample>
  <DurationExample name="m">
    Use this duration when you want an animation to be noticeable without
    distracting the user or slowing them down.
  </DurationExample>
  <DurationExample name="l">
    Use this duration for animations that the user should pay attention to, or
    ones that have a substantial impact to the page.
  </DurationExample>
  <DurationExample name="xl">
    Use this duration for longer animations. It should be used sparingly, since
    users generally don't want to wait this long for animations to complete.
  </DurationExample>
</div>
